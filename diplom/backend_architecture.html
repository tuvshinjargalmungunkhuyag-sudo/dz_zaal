3. Захиалга хийх дараалал
1
Нэвтрэх — Firebase Auth
Хэрэглэгч утасны дугаараа оруулна (+976XXXXXXXX) → Firebase SMS OTP илгээнэ → код баталгаажуулна → UID токен авна
2
Цагийн хуваарь харах
GET /api/schedule?venueId=v1&date=2026-03-24 → Backend Firestore-оос тухайн өдрийн захиалгуудыг шалгана → 08:00–20:00 хүртэл 12 slot буцаана (нээлттэй / захиалагдсан / гэрээт)
3
Захиалга үүсгэх
POST /api/bookings → Backend давхцал шалгана → Давхцал байвал 409 Conflict буцаана → Байхгүй бол Firestore-д хадгална → DBK-XXXXX захиалгын код үүснэ
4
Статус автоматаар шинэчлэгдэх — Cron Job
Сервер дээр 15 минут тутам cron ажиллана → Цаг нь дууссан захиалгуудыг upcoming → completed болгон шинэчилнэ
4. Нэвтрэх дараалал (Firebase Phone Auth)
①
Утасны дугаар оруулах
App +976XXXXXXXX форматаар Firebase-д илгээнэ
②
SMS OTP илгээх

③
Код баталгаажуулах
Хэрэглэгч кодоо оруулна → Firebase баталгаажуулна → UID токен олгоно
④

POST /api/users → Firestore-д хадгалах (утасны дугаар = document ID)

6. Автомат ажиллагаа — Cron Job
⏱️
15 минут тутам автоматаар ажиллана
Сервер дээр тогтмол ажиллах процесс байна. Цагийн бүс: Улаанбаатар (UTC+8)

Дараалал:
1. Firestore-оос status = "upcoming" эсвэл "active" бүх захиалгыг татна
2. Захиалгын дуусах цагийг (dateKey + timeSlotEnd) одоогийн цагтай харьцуулна
3. Хугацаа нь өнгөрсөн захиалгуудыг batch write-аар completed болгон шинэчилнэ
4. completedAt timestamp тэмдэглэнэ
