\section*{\centering \huge{Дүгнэлт}}

Энэхүү бакалаврын төгсөлтийн ажлын хүрээнд Өмнөговь аймгийн Даланзадгад хотын таван спорт заалд зориулсан цаг захиалгын мобайл аппликейшнийг \textbf{Flutter}, \textbf{Node.js/Express}, \textbf{Firebase}, \textbf{Docker}, \textbf{AWS} болон \textbf{AI} технологиудыг хосолсон орчин үеийн стекэд хөгжүүлж, дараахи үр дүнд хүрсэн:

\begin{enumerate}
    \item Даланзадгад хотын таван спорт заалд тулгарч буй цаг захиалгын уламжлалт асуудлуудыг судалж — Говийн Арена, Өмнөговь Спорт Заал, Говийн Волейбол Клуб, Стадионы Волейбол Заал, Цэнтрийн Сагсан Бөмбөгийн Заал — нэг нэгдсэн платформд оруулах шийдэл боловсруулсан. Администраторын самбар байхгүй хялбар архитектурыг сонгосноор апп-ын нарийн төвөгтэй байдал бага байж, хэрэглэгчид шаардлагатай функцэд л анхаарлаа төвлөрүүлсэн.

    \item \textbf{Flutter} фреймворк болон \textbf{Material Design 3} ашиглан говийн цөлийн дулаан тонийг илэрхийлсэн цайвар дизайн (бежийн дэвсгэр \code{\#EDE7DA}, наранжин \code{\#D4700F}, гүн наранжин \code{\#B85C08}, Montserrat фонт) бүхий мобайл аппликейшн хөгжүүлсэн. 4 tab-тай доод навигаци (Нүүр, Захиалгууд, Чат, Профайл), \textbf{StatefulWidget} болон \textbf{setState} суурилсан төлвийн удирдлагыг нэмэлт dependency байхгүйгээр хэрэгжүүлсэн. Апп нь Android болон iOS платформд нэгэн зэрэг ажилладаг.

    \item \textbf{Firebase Authentication}-ийн email/нууц үгийн нэвтрэлтийг хэрэгжүүлж, бүртгэлийн үед \textbf{Nodemailer} ашиглан Gmail SMTP-ээр 6 оронтой баталгаажуулах кодыг хэрэглэгчийн email рүү илгээдэг системийг нэвтрүүлсэн. Код илгээхэд алдаа гарвал Firebase хэрэглэгчийг rollback хийдэг тул хэрэглэгч дахин оролдох боломжтой. \textbf{Cloud Firestore}-д \code{bookings}, \code{users}, \code{fixed\_bookings}, \code{email\_verifications} гэсэн дөрвөн collection тодорхойлж, байгууллагуудын (засгийн газар, компаниудын) долоо хоногийн тогтмол захиалгыг \code{fixed\_bookings} цуглуулгаар хэрэгжүүлсэн. Захиалга бүр \textbf{DBK-xxxxx} форматын давтагдашгүй код авдаг. Цагийн хуваарийн 16 үе (08:00–00:00) тодорхойлогдсон.

    \item \textbf{Node.js/Express} бэкэнд ашиглан REST API хэрэгжүүлж, бэкэнд болон Firebase хоёулаа зэрэгцэн ажиллах гибрид архитектурыг боловсруулсан. \textbf{lib/config.dart} файл нь бэкэндийн URL-ийн нэг эх сурвалж болж хөгжүүлэлтийн болон продакшн орчны ялгааг нэг газраас удирдах боломж олгосон.

    \item \textbf{Docker} контейнержуулалт ашиглан бэкэнд серверийн орчны тогтвортой байдлыг хангаж, \textbf{AWS ECR}-д дүрс хадгалж, \textbf{AWS EC2} (ap-northeast-1, Токио) дээр байршуулсан. \textbf{GitHub Actions} CI/CD хоолой нь \code{backend/} хавтаст өөрчлөлт push хийх бүрт автоматаар Docker дүрс build хийж, ECR-д push хийж, EC2-д шинэ контейнер ажиллуулдаг. Нийт байршуулалт 3--4 минут болно.

    \item Бэкэнд дэх \textbf{cron ажил} нь 15 минут тутамд Firestore-д хандаж дуусгавар болсон захиалгыг \code{completed} төлөвт шилжүүлдэг тул хэрэглэгч захиалга цуцлах шаардлагагүй, систем өөрөө автоматаар шинэчлэгддэг болсон.

    \item \textbf{flutter\_local\_notifications} ашиглан FCM сервергүйгээр захиалгаас нэг цагийн өмнө локал сануулга schedule хийх боломж хэрэгжүүлсэн. Энэ нь серверийн нэмэлт зардал байхгүй, офлайн горимд ч ажилладаг давуу талтай.

    \item AI чатботыг хоёр давхар горимоор хэрэгжүүлсэн: \textbf{Anthropic Claude Haiku 4.5}-г Firebase Cloud Functions дамжуулан үндсэн горим болгож, \textbf{Groq Llama 3.1-8b}-г Express бэкэнд дамжуулан нөөц горим болгосон. Firebase Cloud Functions нь API түлхүүрийг клиент кодод задруулахгүй аюулгүй хадгалах давуу талтай.

    \item Функциональ туршилтаар 10 тест тохиолдол бүгд амжилттай давсан. Гүйцэтгэлийн хэмжилтээр апп 2.3 секундын дотор ачаалж, Firestore хүсэлт 0.6 секунд, EC2 API 0.8 секунд, Claude Haiku 1.4 секунд, Groq Llama 0.9 секундын дотор хариулдаг болохыг баталгаажуулсан.
\end{enumerate}

\textbf{Цаашдын хөгжлийн чиглэл:}

\begin{itemize}
    \item \textbf{Онлайн төлбөр} — QPay, SocialPay зэрэг Монголын төлбөрийн системүүдтэй нэгтгэх;
    \item \textbf{Администраторын функц} — Спорт заалны ажилтнуудад зориулсан хуваарь удирдах дэлгэц нэмэх;
    \item \textbf{Firebase Storage} — Спорт заалнуудын зургийг Firestore-ийн document-тай уялдуулан хадгалах;
    \item \textbf{Firestore-оос заалнуудын мэдээлэл} — Hardcoded жагсаалтыг Firestore collection болгон шилжүүлснээр шинэ заал нэмэх, мэдээлэл засах ажиллагааг хялбар болгох;
    \item \textbf{GPS байршил} — Хэрэглэгчид хамгийн ойрхон заалыг харуулах функц;
    \item \textbf{Статистик самбар} — Захиалгын дүн шинжилгээ, ачаалалтай цагийн мэдээлэл;
    \item \textbf{Өргөтгөл} — Даланзадгад хотоос гадна Өмнөговь аймгийн бусад сумдын спорт байгууламжид өргөтгөх;
    \item \textbf{Flutter Web} — Веб хөтчөөр хандах боломж нэмэх.
\end{itemize}

Энэхүү ажил нь Монгол улсын орон нутгийн хотуудад ухаалаг цахим үйлчилгээ нэвтрүүлэх боломжийг тодорхой харуулж, Flutter болон Firebase-г Node.js/Express REST API, Docker/AWS дэд бүтэц, GitHub Actions CI/CD автоматжуулалт, AI чатбот үйлчилгээтэй хосолсон орчин үеийн бүрэн стекийн туршлагыг цуглуулсан чухал ажил болсон гэж дүгнэж байна. Ажлын явцад олж авсан туршлага нь цаашид ижил төстэй орон нутгийн цахим үйлчилгээний системүүдийг хөгжүүлэхэд хэрэглэж болохуйц загвар болно.
