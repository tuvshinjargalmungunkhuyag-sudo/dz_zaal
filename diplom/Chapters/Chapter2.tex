Flutter-ийн \textbf{Skia/Impeller} дүрслэлийн хөдөлгүүр нь бүх UI-г платформаас үл хамааран өөрөө зурдаг тул Android болон iOS-д яг ижил харагдах баталгаа олгодог нь энэ ажлын хувьд чухал давуу тал юм.

\subsection{Flutter-ийн тухай}

\keyword{Flutter} нь Google компанийн 2018 онд гаргасан нээлттэй эхийн UI фреймворк юм. Flutter-ийн гол давуу талд дараахи зүйлс орно:

\begin{itemize}
    \item \textbf{Hot Reload} — Кодын өөрчлөлтийг апп-г дахин эхлүүлэхгүйгээр шууд харах боломж;
    \item \textbf{Widget-д суурилсан архитектур} — UI бүрэлдэхүүн хэсэг болгоныг widget хэлбэрээр тодорхойлно;
    \item \textbf{Skia/Impeller render engine} — Бүх UI-г платформаас үл хамааран өөрөө зурдаг тул платформ бүрт ижил харагдах баталгаа олгодог;
    \item \textbf{Нэг кодын бааз} — Android, iOS, Web, Desktop платформд нэгэн зэрэг гаргах боломж.
\end{itemize}

Flutter нь version 3.x хүрэхдээ Material Design 3 (Material You) стандартыг бүрэн дэмждэг болж, \code{ThemeData} классаар дамжуулан дүрс, өнгө, хэлбэр бүх зүйлийг системтэй тохируулах боломж нээгдсэн \cite{material3}. Энэхүү ажилд Flutter 3.x хувилбарыг ашигласан.

\subsection{Dart програмчлалын хэл}

\keyword{Dart} нь Google-ийн боловсруулсан, object-oriented, strongly typed хэл юм. Flutter хөгжүүлэлтэд Dart ашигладаг бөгөөд дараахи онцлогтой \cite{dart_doc}:

\begin{itemize}
    \item \textbf{Async/Await} — Асинхрон үйлдлийг хялбараар бичих, Future болон Stream дэмждэг;
    \item \textbf{Null Safety} — Null pointer алдааг compile time-д илрүүлэх;
    \item \textbf{AOT ба JIT compile} — Хурдан хөгжүүлэлт (JIT) болон хурдан гүйцэтгэл (AOT) нэгэн зэрэг хангах;
    \item \textbf{Stream} — Бодит цагийн өгөгдөл боловсруулах, Firebase Firestore-ийн real-time listener-тэй холбоход тохиромжтой.
\end{itemize}

Dart хэлний \code{class} бүтэц нь Flutter-ийн widget системтэй нягт уялддаг. \code{StatelessWidget} нь өөрчлөлтгүй UI-г илэрхийлж, \code{StatefulWidget} нь дотоод \code{State} объектоор дамжуулан өөрчлөгддөг UI-г удирддаг.

\subsection{StatefulWidget ба setState-д суурилсан төлвийн удирдлага}

Flutter апп-д төлвийн удирдлагыг (state management) зөв хийх нь апп-ын найдвартай байдал, гүйцэтгэлд шууд нөлөөлнө. Энэ ажилд \keyword{StatefulWidget} болон \keyword{setState} хослолыг ашигласан. Энэ нь Flutter-ийн бүтэц дэх хамгийн анхдагч, ойлгомжтой механизм бөгөөд нэмэлт багц шаарддаггүй.

Provider, Riverpod зэрэг өндөр түвшний төлвийн удирдлагын хэрэгслүүдтэй харьцуулахад \code{setState} нь дараахи тохиолдолд тохиромжтой:

\begin{itemize}
    \item Апп-ын нарийн төвөгтэй байдал тодорхой хэмжээнд байх;
    \item Шинэ хөгжүүлэгчдэд ойлгомжтой, уншиж болохуйц байх шаардлагатай;
    \item Нэмэлт dependency нэмэхгүйгээр апп-ын хэмжээг жижиг байлгах.
\end{itemize}

\begin{lstlisting}[language={}, caption={StatefulWidget ба setState жишээ}]
class BookingScreen extends StatefulWidget {
  const BookingScreen({super.key});
  @override
  State<BookingScreen> createState() => _BookingScreenState();
}

class _BookingScreenState extends State<BookingScreen> {
  String? selectedTimeSlot;
  bool isLoading = false;

  void _selectSlot(String slot) {
    setState(() {
      selectedTimeSlot = slot;
    });
  }
  // ...
}
\end{lstlisting}

Апп-ын ерөнхий навигацийг \code{BottomNavigationBar} widget дамжуулан 4 tab-тай хэрэгжүүлсэн: Нүүр (Home), Захиалгууд (Bookings), Чат (Chat), Профайл (Profile). Идэвхтэй tab-ийн индексийг root StatefulWidget-ийн \code{setState}-д хадгалдаг.

\section{Firebase бэкэнд үйлчилгээ}

\keyword{Firebase} нь Google-ийн санал болгодог BaaS (Backend as a Service) платформ юм. Уг платформ нь мобайл апп хөгжүүлэгчдэд бэкэнд дэд бүтэц тохируулах шаардлагагүйгээр өгөгдлийн сан, нэвтрэлт зэрэг үйлчилгээг ашиглах боломж олгодог \cite{firebase_doc}.

\subsection{Cloud Firestore}

\keyword{Cloud Firestore} нь NoSQL, баримт жишгийн (document-oriented) өгөгдлийн сан юм \cite{firestore_doc}. Бодит цагийн шинэчлэлт (real-time listeners), өгөгдлийг offline хадгалах, автомат масштабчлал зэрэг онцлогтой. Firestore нь \textbf{Collection} болон \textbf{Document} бүтэцтэй:

\begin{itemize}
    \item \textbf{Collection} — Баримтуудын цуглуулга (энэ ажилд: \code{users}, \code{bookings}, \code{fixed\_bookings});
    \item \textbf{Document} — Талбаруудын цуглуулга, JSON форматтай төстэй.
\end{itemize}

Firestore-ийн security rules нь collection бүрт хэрэглэгч хандах эрхийг нарийн тохируулах боломж олгодог. Энэ ажилд нэвтрэлт хийсэн хэрэглэгч зөвхөн өөрийн захиалгыг унших, шинэ захиалга бичих эрхтэй.

\subsection{Firebase Authentication}

\keyword{Firebase Authentication} нь хэрэглэгчийн нэвтрэлтийг аюулгүй удирдах үйлчилгээ юм \cite{firebase_auth}. Энэ ажилд \textbf{email хаяг болон нууц үгийн} нэвтрэлтийг ашигласан. Хэрэглэгч анх бүртгүүлэх үед Firebase нь \code{createUserWithEmailAndPassword} функцээр UID үүсгэж, дараа нь Node.js бэкэнд Nodemailer-ийн тусламжтайгаар тухайн email хаягт \textbf{6 оронтой баталгаажуулах код} илгээнэ. Хэрэглэгч кодоо амжилттай оруулсны дараа \code{emailVerified: true} болж нэвтрэх эрх нээгддэг.

Нэвтрэх үед Firebase-ийн \code{signInWithEmailAndPassword} функцийг ашигладаг бөгөөд Firestore-д хадгалсан \code{emailVerified} талбарыг шалгаж, баталгаажаагүй хэрэглэгчийн нэвтрэлтийг хориглодог. Энэ нь email хаягийн үнэн зөвийг баталгаажуулахын зэрэгцээ аюулгүй байдлыг нэмэгдүүлдэг.

\subsection{Firebase Cloud Functions}

\keyword{Firebase Cloud Functions} нь серверлэс (serverless) тооцооллын үйлчилгээ юм. Энэ ажилд Cloud Functions-ийг AI чатботын үндсэн endpoint болгон ашигласан: Flutter апп Cloud Function руу хүсэлт илгээхэд уг функц Anthropic-ийн API руу дамжуулан Claude Haiku хариулт авч буцаана. Cloud Functions нь серверийн дэд бүтцийг өөрөө удирддаг тул хөгжүүлэгч апп-ын логикт төвлөрөх боломжтой болдог.

\section{Node.js болон Express.js бэкэнд}

\subsection{Node.js}

\keyword{Node.js} нь Chrome V8 JavaScript engine дээр суурилсан серверийн талын JavaScript ажиллуулах орчин юм \cite{nodejs_doc}. Node.js-ийн гол давуу талууд:

\begin{itemize}
    \item \textbf{Event-driven, non-blocking I/O} — Олон нэгэн зэрэг холболтыг цөөн нөөцөөр боловсруулах чадвартай;
    \item \textbf{npm экосистем} — Дэлхийн хамгийн том нээлттэй эхийн багцын сан;
    \item \textbf{JavaScript нэгдсэн байдал} — Клиент болон серверийн талд нэг хэл ашиглах боломж;
    \item \textbf{Хурдан хөгжүүлэлт} — JSON боловсруулалт, REST API бүтээхэд маш тохиромжтой.
\end{itemize}

\subsection{Express.js фреймворк}

\keyword{Express.js} нь Node.js дээр суурилсан хөнгөн, уян хатан веб фреймворк юм \cite{expressjs_doc}. RESTful API барихад хамгийн өргөн хэрэглэгддэг Node.js фреймворк бөгөөд дараахи онцлогтой:

\begin{itemize}
    \item \textbf{Middleware систем} — Хүсэлт боловсруулах дамжлагуудыг зохион байгуулах;
    \item \textbf{Router} — URL хаягийн чиглүүлэлтийг модуль болгон хуваах;
    \item \textbf{Хөнгөн бүтэц} — Шаардлагатай зүйлийг л ачаалдаг.
\end{itemize}

\subsection{REST API архитектур}

\keyword{REST} (Representational State Transfer) нь Roy Fielding (2000) \cite{rest_fielding} боловсруулсан веб үйлчилгээний архитектурын хэв маяг юм. REST API нь HTTP протоколын GET, POST, PUT, DELETE арга ашиглан нөөцийг удирддаг. REST-ийн гол шинж чанарууд: \textit{Stateless} (сервер хүсэлт бүрийг бие даасан гэж үздэг), \textit{Uniform Interface} (стандарт URL + HTTP арга), \textit{Resource-based} (нөөц бүрийг URL-ээр илэрхийлнэ). Энэ ажилд Flutter апп болон Node.js бэкэнд хоорондын харилцааг RESTful API-аар гүйцэтгэдэг бөгөөд өгөгдлийн солилцоог JSON форматаар хийдэг.

\section{Docker контейнержуулалт}

\keyword{Docker} нь аппликейшн болон түүний бүх хамаарлыг тусгаарлагдсан контейнерт багцалдаг платформ юм \cite{docker_doc}. Docker ашиглахын давуу тал:

\begin{itemize}
    \item \textbf{Тогтвортой орчин} — Хөгжүүлэлтийн болон продакшн орчин яг ижил байх баталгаа;
    \item \textbf{Хурдан байршуулалт} — Контейнер хэдхэн секундэд эхлэнэ;
    \item \textbf{Тусгаарлалт} — Системийн бусад хэсгийг нөлөөлөхгүй;
    \item \textbf{Хэмжээс (Scale)} — Хэрэгцээ нэмэгдэхэд хялбар масштабчлал.
\end{itemize}

\textbf{Dockerfile} нь апп-ын дүрсийг (image) тодорхойлдог тохиргооны файл юм. Express бэкэндийн Dockerfile нь Node.js суурь дүрс дээр \code{package.json}-г суулгаж, эх кодыг хуулан, 3000 портоор сервер эхлүүлдэг.

\begin{lstlisting}[language={}, caption={Node.js бэкэндийн Dockerfile}]
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
\end{lstlisting}

\section{AWS дэд бүтэц}

\subsection{Amazon EC2}

\keyword{Amazon EC2} (Elastic Compute Cloud) нь AWS-ийн виртуал сервер (instance) үйлчилгээ юм \cite{aws_ec2_doc}. Энэ ажилд Node.js/Express бэкэнд сервер нь \textbf{ap-southeast-1} (Сингапур) бүсийн EC2 instance дээр ажилладаг. EC2 нь дараахи давуу талтай:

\begin{itemize}
    \item Серверийн тохиргоог бүрэн хянах боломж;
    \item Docker контейнер ажиллуулах дэмжлэг;
    \item Security Group-ийн тусламжтай сүлжээний хандалтыг нарийн удирдах.
\end{itemize}

\subsection{Amazon ECR}

\keyword{Amazon ECR} (Elastic Container Registry) нь AWS-ийн Docker дүрсний бүртгэлийн үйлчилгээ юм \cite{aws_ecr_doc}. GitHub Actions-аар build хийсэн Docker дүрсийг ECR-д push хийж, EC2 instance тэр дүрсийг татан ажиллуулдаг. Энэ нь дүрсний нийцтэй удирдлага, нууцлалыг хангадаг.

\section{GitHub Actions CI/CD}

\keyword{GitHub Actions} нь GitHub платформ дахь CI/CD (Continuous Integration / Continuous Deployment) автоматжуулалтын хэрэгсэл юм \cite{github_actions_doc}. Workflow нь YAML файлаар тодорхойлогддог бөгөөд тодорхой нөхцөл (жишээ: \code{backend/} хавтасны файл өөрчлөгдөх) биелэхэд автоматаар ажилладаг.

Энэ ажлын CI/CD хоолой нь дараахи алхмуудтай:

\begin{enumerate}
    \item Хөгжүүлэгч \code{backend/} хавтаст өөрчлөлт хийж GitHub-д push хийнэ;
    \item GitHub Actions trigger ажиллана;
    \item Node.js бэкэнд кодоос Docker дүрс build хийнэ;
    \item Build хийсэн дүрсийг AWS ECR-д push хийнэ;
    \item SSH-аар AWS EC2 instance-д холбогдож ECR-аас шинэ дүрсийг pull хийнэ;
    \item Хуучин контейнерийг зогсоож, шинэ контейнерийг ажиллуулна.
\end{enumerate}

GitHub Actions нь AWS-ийн нэвтрэлтийн нууцлал (access key, secret), EC2-ийн SSH түлхүүр зэргийг GitHub Secrets-д хадгалан аюулгүй ашигладаг.

\section{AI үйлчилгээнүүд}

\subsection{Anthropic Claude Haiku}

\keyword{Anthropic Claude Haiku 4.5} нь Anthropic компанийн хурдан, хэмнэлттэй LLM загвар юм \cite{anthropic_doc}. Claude Haiku нь богино хариулт өгдөг асуудлуудад маш хурдан ажиллах бөгөөд API дамжуулан ашиглах боломжтой. Энэ ажилд Firebase Cloud Functions дотор Anthropic-ийн SDK-г ашиглан спорт заалтай холбоотой асуултуудад хариулдаг AI чатбот хэрэгжүүлсэн. Cloud Functions нь API түлхүүрийг Flutter кодод задруулахгүйгээр аюулгүй хадгалах боломж олгодог.

\subsection{Groq Llama 3.1-8b}

\keyword{Groq} нь LPU (Language Processing Unit) процессор ашиглан LLM загваруудыг асаар хурдан ажиллуулдаг платформ юм \cite{groq_doc}. Groq дээрх \textbf{Llama 3.1-8b} загварыг Node.js/Express бэкэнд дамжуулан нөөц горим болгон ашигласан. Хэрэв Firebase Cloud Functions-ийн Claude хариулт удаан болох буюу алдаа гарвал Flutter апп \code{lib/config.dart}-д тодорхойлсон бэкэнд URL-аар Groq API-руу хандана.

\section{flutter\_local\_notifications}

\keyword{flutter\_local\_notifications} нь Firebase Cloud Messaging (FCM) ашиглахгүйгээр төхөөрөмж дотор мэдэгдэл (local notification) харуулах Flutter багц юм \cite{local_notifications_doc}. Мобайл аппликейшнд push notification хэрэгжүүлэхэд ихэвчлэн FCM сервер шаарддаг боловч энэ багц нь сервергүйгээр хэрэглэгчийн төхөөрөмж дотор мэдэгдэл шедуль хийх боломжтой.

Энэ ажилд \code{flutter\_local\_notifications}-ийг дараах зорилгоор ашигласан:

\begin{itemize}
    \item \textbf{1 цагийн өмнөх сануулга} — Хэрэглэгч цаг захиалах үед Flutter апп тухайн цагаас 1 цаг хасаж (\code{scheduledDate = bookingTime.subtract(Duration(hours: 1))}) сануулгын мэдэгдлийг schedule хийнэ;
    \item \textbf{Offline найдвартай байдал} — Интернет холболт байхгүй тохиолдолд ч гэсэн төхөөрөмж дотор хадгалагдсан мэдэгдэл ажилладаг;
    \item \textbf{Хоёр платформ} — Android болон iOS аль алинд дэмжлэгтэй, тохиргоо нэг кодоор хийгддэг;
    \item \textbf{Нэмэлт зардалгүй} — FCM сервер байхгүй тул серверийн нэмэлт зардал шаарддаггүй.
\end{itemize}

\section{Ижил төстэй системүүдийн дүн шинжилгээ}

\subsection{Монгол улсын цаг захиалгын системүүд}

Монгол улсын Google Play дээрх судалгаанаас спортын байгууламжийн захиалгатай холбоотой дараах системийг илрүүлж, дүн шинжилгээ хийсэн:

\textbf{eZaal} (DAZO LLC, Монгол) нь Монгол улсын спорт заал, фитнес төв, хөлбөмбөгийн талбай, бүжгийн студи зэрэг олон спортын байгууламжийг нэгтгэсэн агрегатор платформ юм \cite{ezaal_app}. Google Play дээр \textbf{10,000+} удаа татагдсан, хэрэглэгчийн үнэлгээ \textbf{4.0/5.0} бөгөөд хамгийн сүүлийн шинэчлэлт 2025 оны 6 дугаар сард хийгдсэн (хувилбар 2.2.1). Байгууламжийн зураг, үнэ тариф, цагийн хуваарь, байршлын мэдээлэл харуулах, QPay интеграцаар онлайн төлбөр хийх, нэмэлт шимтгэлгүйгээр захиалга хийх боломжтой. Гэхдээ зөвхөн Android платформд зориулагдсан (iOS хувилбар байхгүй), AI чатбот болон CI/CD автоматжуулалт байхгүй, Улаанбаатар болон томоохон хотуудад чиглэсэн тул Даланзадгад зэрэг орон нутгийн хотуудад үйлчилгээ хүрдэггүй. Байгуулагын тогтмол захиалгын тусгай удирдлага болон гурван давхаргат үүлэн архитектур хэрэгжүүлээгүй.
