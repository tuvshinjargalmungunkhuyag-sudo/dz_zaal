\section{Туршилтын орчин, аргачлал}

Системийг гүйцэд хөгжүүлсний дараа функциональ болон гүйцэтгэлийн туршилтыг гурван үе шаттайгаар явуулсан. Туршилт нь \textbf{функциональ}, \textbf{нэгтгэлийн} болон \textbf{хэрэглэхэд хялбар байдлын} гэсэн гурван хэсгээс бүрдсэн.

Туршилтад ашигласан техник хэрэгсэл:

\begin{itemize}
    \item Android 13 (Samsung Galaxy A54) болон iOS 16 (iPhone 13) гар утас;
    \item AWS EC2 t3.micro сервер, Docker контейнер (ap-northeast-1 бүс);
    \item Firebase Cloud Firestore (us-central1);
    \item 4G LTE болон Wi-Fi сүлжээ.
\end{itemize}

Туршилтын нийт тохиолдлын тоо: функциональ туршилт 10, нэгтгэлийн туршилт 4, гүйцэтгэлийн хэмжилт 6, нийт \textbf{20} тохиолдол.

\subsection{Байгууллагын тогтмол захиалгын туршилт}

Засгийн газрын байгууллагуудын долоо хоногт давтагдах захиалгыг \code{fixed\_bookings} цуглуулгаас уншиж цагийн хуваарьт тусгадаг. Энэ функцийг дараахи байдлаар шалгасан:

Тогтмол захиалгатай цаг нь нил ягаан өнгөөр тэмдэглэгдэж ``Гэрээт'' гэсэн шошго харагдсан. Тэр цагт хэрэглэгч захиалга хийхийг оролдоход товч дарагдахгүй хаагддаг нь зөв ажилласан. Цаг хуваарийн дэлгэц нэг харагдахуйц байж байгаа нь туршилтын явцад сайн тал болж ажиглагдсан.

\section{Аюулгүй байдлын шалгалт}

Системийн гол аюулгүй байдлын тохиргоонуудыг шалгасан:

\begin{itemize}
    \item \textbf{Email/нууц үгийн нэвтрэлт} — Firebase Authentication нь нэг email хаягаас хэт олон нэвтрэх оролдлого ирэхэд автоматаар хааж хамгаалдаг; 6 оронтой баталгаажуулах код 10 минутын дараа хүчингүй болдог тул нэвтрэлтийн аюулгүй байдлыг нэмэгдүүлнэ;

    \item \textbf{API түлхүүрүүдийн хадгалалт} — AWS болон Groq-ийн нэвтрэлтийн түлхүүрүүд GitHub Secrets болон \code{.env} файлд хадгалагдаж, апп-ын кодонд нэг ч газар задраагүй болохыг шалгасан;

    \item \textbf{Firestore дүрэм} — \code{bookings} цуглуулгад бичих хүсэлт нь зөвхөн нэвтэрсэн хэрэглэгчдэд зөвшөөрөгдөхөөр тохируулагдсан болохыг \code{firebase emulator}-оор шалгасан;

    \item \textbf{HTTP cleartext traffic} — Flutter апп болон EC2 хоорондын холболт HTTP дээр явагдах тул Android 9+-д \code{android:usesCleartextTraffic="true"} тохиргоог \code{AndroidManifest.xml}-д нэмж шийдвэрлэсэн. Цаашид SSL/HTTPS тохируулах нь аюулгүй байдлын оновчтой шийдэл болно.
\end{itemize}
