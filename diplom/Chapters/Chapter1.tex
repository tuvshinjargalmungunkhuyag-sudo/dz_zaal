\section*{\centering \Large УДИРТГАЛ}

\subsection*{Сэдэв сонгосон үндэслэл}

Мэдээллийн технологийн хурдацтай хөгжил нь орчин үеийн иргэдийн өдөр тутмын амьдралыг ихээхэн өөрчилж байна. Цахим захиалга, онлайн үйлчилгээ нь дэлхийн өнцөг булан бүрт нэвтэрч, хүмүүсийн цаг хугацааг хэмнэж, үйлчилгээний чанарыг дээшлүүлж байна. Statista судалгааны байгууллагын 2024 оны тайланд дурдснаар дэлхийн мобайл аппликейшний зах зээлийн хэмжээ 2028 он гэхэд 756 тэрбум ам.долларт хүрнэ гэж таамаглаж байна \cite{statista_market}. Гэвч Монгол улсын орон нутгийн хотуудад, тухайлбал Өмнөговь аймгийн Даланзадгад хотод, уламжлалт үйлчилгээний хэлбэр давамгайлсаар байна.

Даланзадгад хотод өнөөдрийн байдлаар таван спорт заал болон тамирын цогцолбор үйл ажиллагаа явуулж байна: Өмнөговь аймгийн Спорт цогцолбор, Говийн Арена Спорт заал, 1-р хорооны Спорт заал, 2-р хорооны Спорт заал, Даланзадгад Волейболын клуб заал. Эдгээр байгууламж нь аймгийн иргэдийн биеийн тамирын үйл ажиллагааны гол бааз болдог.

Иргэд эдгээр заалд цаг захиалахдаа тулгамдаж буй асуудлууд болон тэдгээрийн шийдлийг доорх хүснэгтэд үзүүлэв.

\subsection*{Зорилго}

Энэхүү ажлын зорилго нь Өмнөговь аймгийн Даланзадгад хотын таван спорт байгууламжийн цаг захиалгыг цахимжуулж, хэрэглэгчдэд хялбар, хурдан, ил тод байдлаар захиалга хийх боломж олгох \keyword{Flutter} мобайл аппликейшн болон \keyword{Node.js/Express} бэкэнд бүхий гурван давхаргат системийг үүлэн орчинд хөгжүүлж, байршуулах явдал юм.

%-------------------------------------------------------------------------------

\subsection*{Зорилт}

Уг зорилгод хүрэхийн тулд дараах зорилтуудыг дэвшүүлж байна. Нэгдүгээрт, Даланзадгад хотын спорт заалнуудын цаг захиалгын одоогийн нөхцөл байдал, иргэдийн хэрэгцээг судлаж, тулгамдаж буй асуудлуудыг тогтооно. Хоёрдугаарт, цаг захиалгын мобайл аппликейшний функциональ болон функциональ бус шаардлагын дүн шинжилгээ хийж, Use Case Diagram, Activity Diagram, Sequence Diagram боловсруулна. Гуравдугаарт, Flutter болон Firebase технологийг ашиглан гурван давхаргат системийн архитектур зохион байгуулна. Дөрөвдүгээрт, Material Design 3 стандартад нийцсэн мобайл аппликейшн хэрэгжүүлнэ. Тавдугаарт, Node.js/Express REST API бэкэнд хөгжүүлж, Docker контейнержуулалтаар AWS EC2-д байршуулна. Зургадугаарт, GitHub Actions CI/CD хоолойгоор бэкэндийн автоматжуулсан байршуулалтыг хэрэгжүүлнэ. Долоодугаарт, функциональ, нэгтгэлийн болон гүйцэтгэлийн туршилт хийж, хэрэглэхэд хялбар байдлыг үнэлнэ.

%-------------------------------------------------------------------------------

\subsection*{Судлагдсан байдал}

Спорт заал болон үйлчилгээний цаг захиалгын мобайл аппликейшн хөгжүүлэлтийн чиглэлд олон улсын болон дотоодын судлаачид тодорхой хэмжээний ажил хийсэн байна.

Монгол улсын хэмжээнд eZaal (DAZO LLC) нь спортын байгууламжийн захиалгыг нэгтгэсэн томоохон платформ бөгөөд Google Play дээр 10,000 гаруй удаа татагдсан \cite{ezaal_app}. Kassim et al. (2022) судалгаанд Flutter-г ашиглан sport facility booking систем хөгжүүлсэн туршлагыг дүрсэлсэн бөгөөд Firebase-тэй хослуулсан нь хөгжүүлэлтийн хугацааг 40 хувиар бууруулсан гэж тэмдэглэжээ \cite{flutter_booking}.

Монголын эрдэм шинжилгээний хэмжээнд Амарсайханы Билгүүн \cite{thesis_mobile} мобайл аппликейшн хөгжүүлэлтийн аргачлалын асуудлыг судалсан бол Т.Амартүвшин, Б.Зоригтбаатар нар \cite{mobile_mongol} мобайл апп хөгжүүлэлтийн онолын үндсийг тодорхойлсон байна. Гэхдээ тухайлан үүлэн орчинд суурилсан, гурван давхаргат архитектур бүхий спорт заалны цаг захиалгын тусгай системийн талаарх судалгаа орон нутгийн хэмжээнд хийгдээгүй байна. Иймд энэхүү ажил нь орон нутгийн хэрэгцээнд нийцсэн, cross-platform, AI чатбот болон CI/CD автоматжуулалт бүхий орчин үеийн технологийн бүрэн стек ашигласан шинэлэг шийдлийг санал болгож байна.

\subsection*{Шинэлэг тал}

Энэхүү ажлын шинэлэг тал нь дараахид оршино. Нэгдүгээрт, Flutter клиент, Node.js/Express бэкэнд (AWS EC2), Firebase/Google Cloud дэд бүтэц гэсэн тодорхой давхаргаар зохион байгуулсан гурван давхаргат үүлэн архитектурыг хэрэгжүүлсэн. Хоёрдугаарт, GitHub Actions ашиглан код шинэчлэгдэх бүрт Docker дүрс build хийж, AWS ECR-д push хийж, EC2-д автоматаар байршуулдаг 3--4 минутын CI/CD хоолой бий болгосон. Гуравдугаарт, Anthropic Claude Haiku 4.5 болон Groq Llama 3.1-8b загваруудыг нөөцлөлтийн горимоор нэгтгэсэн давхар горимын AI чатбот хэрэгжүүлсэн. Дөрөвдүгээрт, Даланзадгад хотын байгууллагуудын долоо хоногийн тогтмол цагийг удирдах тусгай \code{fixed\_bookings} функц нэмсэн. Тавдугаарт, Монгол улсын орон нутгийн хотод анх удаа cross-platform, AI чатбот бүхий спорт заалны мобайл захиалгын систем хэрэгжүүлсэн.

\subsection*{Судалгааны хамрах хүрээ}

Энэхүү ажлын хүрээнд иргэдэд зориулсан мобайл апп (Android 8.0+ болон iOS 13.0+), Firebase дээр суурилсан email/нууц үгийн нэвтрэлт (Nodemailer 6 оронтой кодын баталгаажуулалттай), Node.js/Express REST API бэкэнд (5 endpoint), Docker контейнержуулалт болон AWS EC2/ECR байршуулалт (ap-southeast-1), GitHub Actions CI/CD автоматжуулалт, AI чатботын үйлчилгээ (Claude Haiku 4.5 болон Groq Llama 3.1-8b), мөн локал мэдэгдлийн систем орно. Онлайн төлбөрийн систем (QPay, SocialPay), GPS байршлын нарийн функц болон администраторын дэлгэц нь энэ ажлын хамрах хүрээнд ороогүй болно.

\subsection*{Ач холбогдол}

Энэхүү ажил нь Даланзадгад хотын иргэдэд спорт заал захиалах үйл явцыг хялбарчилж, цаг хугацааг хэмнэнэ. Спорт заалны ажилтнуудад захиалгын менежментийг автоматжуулж, ажлын ачааллыг бууруулна. Цахим захиалга нэвтэрснээр буруу ойлголцол болон шугам завгүй байхтай холбоотой асуудлуудыг арилгаж, захиалгын ил тод байдлыг хангана. Мөн энэ ажил нь орон нутгийн хотуудад цахим үйлчилгээ нэвтрүүлэх практик жишиг болж, цаашид ижил төрлийн шийдлүүдэд тулгуур болох ач холбогдолтой.

\subsection*{Ажлын бүтэц}

Энэхүү бакалаврын төгсөлтийн ажил дараахи бүтэцтэй.

Удиртгалд судалгааны үндэслэл, зорилго, зорилт, судлагдсан байдал, шинэлэг тал, хамрах хүрээ, ач холбогдлыг тодорхойлсон.

1-р бүлэгт (Онол ба технологийн судалгаа) гурван тулгуур асуудлыг онолын үндэслэлтэйгээр авч үзсэн. 1.1 дэд бүлэгт мобайл хөгжүүлэлтийн технологи --- Flutter, Dart, Firebase, Material Design 3 --- яагаад сонгосон, хэрхэн ашигласан болохыг тайлбарласан. 1.2 дэд бүлэгт бэкэнд хөгжүүлэлтийн технологи --- Node.js, Express, REST API, AI чатбот --- -ийн онолын үндэс болон системд гүйцэтгэх үүргийг тодорхойлсон. 1.3 дэд бүлэгт үүлэн орчин ба автоматжуулалт --- Docker, AWS EC2/ECR, GitHub Actions CI/CD --- хэрэгтэй болсон шалтгаан, хэрэглэх аргачлалыг судалсан.

2-р бүлэгт (Системийн загварчлал ба хөгжүүлэлт) системийг зохион байгуулж, хэрэгжүүлсэн байдлыг дөрвөн дэд бүлгээр нарийвчлан тайлбарласан. 2.1 дэд бүлэгт функциональ болон функциональ бус шаардлагын дүн шинжилгээ, Use Case, Activity, Sequence диаграммуудыг боловсруулсан. 2.2 дэд бүлэгт Flutter аппликейшний дэлгэцүүд, навигаци, хэрэглэгчийн интерфейсийн хэрэгжүүлэлтийг харуулсан. 2.3 дэд бүлэгт Node.js/Express REST API болон email баталгаажуулалтын систем, AI чатботын хэрэгжүүлэлтийг тайлбарласан. 2.4 дэд бүлэгт Firestore өгөгдлийн сангийн бүтэц, collection-уудын загварыг харуулсан.

3-р бүлэгт (Хэрэгжүүлэлт) системийг бодит орчинд байршуулж, туршсан үр дүнг тайлбарласан. 3.1 дэд бүлэгт Docker контейнержуулалт, AWS EC2 байршуулалт, GitHub Actions CI/CD хоолойн хэрэгжүүлэлтийг дэлгэрэнгүй харуулсан. 3.2 дэд бүлэгт функциональ туршилт (шаардлагатай холбосон), нэгтгэлийн туршилт, гүйцэтгэлийн хэмжилт, хэрэглэхэд хялбар байдлын үнэлгээний үр дүнг нэгтгэсэн.

Дүгнэлтэд ажлын нийт үр дүн, хүрсэн зорилтуудын биелэлт болон цаашдын хөгжлийн чиглэлийг дүгнэсэн.
